# Chapter 0: The Starting Line

## What You're About to Build

Ten lines of code. That's Chapter 1.

By Chapter 5, your agent reads files, runs shell commands, and edits code. By Chapter 9, it rewrites its own behavior based on what worked and what didn't. By Chapter 15, it's a swarm — multiple agents discovering each other, passing messages, self-organizing around tasks.

But right now? You just need a working LLM call and a project directory. Let's get you there.

---

## Get an API Key

Your agent's brain is an LLM. You need API access to one. Pick whichever you're comfortable with — or whichever is cheapest today. The book works with any of them.

**Anthropic (Claude) — recommended.** Go to [console.anthropic.com](https://console.anthropic.com), create an account, generate an API key. Claude has strong tool-calling and structured output, which matters in later chapters.

**Google (Gemini) — strong alternative.** Go to [aistudio.google.com](https://aistudio.google.com), create an API key. Gemini offers a generous free tier and solid tool-use support.

**OpenAI — alternative.** Go to [platform.openai.com](https://platform.openai.com), same deal. GPT-4o and newer models work well for everything in this book.

**Ollama — free, local, no key needed.** Install from [ollama.com](https://ollama.com), pull a model (`ollama pull llama3`), and you're running locally. Slower and less capable, but it works, costs nothing, and your data never leaves your machine.

**Something else?** Any LLM with a chat completions API and tool-calling support will work. Mistral, Groq, Together, Fireworks — pick your favorite. If it can take a system prompt, hold a conversation, and call functions, you're good. tbh, the best way to learn is with the LLM you already know.

Whichever you pick, store the key in an environment variable. Not in code. Not in a config file that gets committed. In your shell:

```bash
# Pick the ones you need
export ANTHROPIC_API_KEY="sk-ant-..."
export GOOGLE_API_KEY="AI..."
export OPENAI_API_KEY="sk-..."
```

Add it to your `.bashrc` / `.zshrc` so it persists.

Prefer a project-local pattern? Commit a `.env.example` template with empty values, gitignore the real `.env`. Anyone cloning your project gets the blanks to fill in; your key never leaves your machine.

```
# .env.example — commit this. The real .env is gitignored.
ANTHROPIC_API_KEY=
GOOGLE_API_KEY=
OPENAI_API_KEY=
```

Load it with `source .env` before running, or let `python-dotenv` pick it up at import time. Same rule either way: the template is shared, the secret isn't.

## Set Up Python

We're using Python. Not because it's the best language — because it has the least boilerplate between you and the LLM. Three lines to make an API call. No build step. No type ceremony.

> **The specs in this book are language-agnostic pseudocode.** If you prefer Go, Rust, or TypeScript, everything translates — the interfaces are the same. We use Python because it gets out of the way.

You need Python 3.10+. Check:

```bash
python3 --version
# Python 3.10.x or higher
```

Create your project:

```bash
mkdir tbh-code && cd tbh-code
python3 -m venv .venv
source .venv/bin/activate
```

Install the LLM abstraction layer:

```bash
pip install 'aisuite[anthropic]'
# or swap the provider: aisuite[google], aisuite[openai], aisuite[ollama]
# or install everything: pip install 'aisuite[all]'
```

One library. [aisuite](https://github.com/andrewyng/aisuite) gives you a single `client.chat.completions.create()` call that works with every major provider — Anthropic, Google, OpenAI, Ollama, Mistral, Groq, and more. Same input format. Same output format. You pick the provider at init time; the rest of your code never changes.

We use aisuite for exactly one thing: **making LLM calls without provider-specific code.** That's it. Everything else — the agent loop, tool calling, MCP integration, memory, planning, the swarm — you build yourself. That's the whole point of this book. aisuite handles the boring part (API differences) so you can focus on the interesting part (agent architecture).

> **Alternative:** If you need 100+ providers or a proxy gateway, [LiteLLM](https://github.com/BerriAI/litellm) works the same way. Everything in this book is compatible with either.

## The Todo-API Codebase

Throughout this book, your agent works against a single codebase: **todo-api**. It's a REST API for managing todos — auth, tasks, models, tests. About 10 files of pseudocode.

It lives in the book's repo at `spec/todo-api/`. Clone the repo and copy it into your project:

```bash
# from inside tbh-code/
cp -r /path/to/book-repo/spec/todo-api ./todo-api
```

Why pseudocode? Because `todo-api` isn't something you run — it's something your agent *reads and reasons about*. It has intentionally planted bugs. Across chapters, your agent will find them, explain them, and eventually fix them. The bugs are the curriculum.

## Prove It Works

Create `smoke_test.py` in your project root. Pick your model string — everything else is identical:

```python
# smoke_test.py
import aisuite as ai

client = ai.Client()

# Pick ONE. Uncomment the model you want.
MODEL = "anthropic:claude-sonnet-4-20250514"   # Claude
# MODEL = "google:gemini-2.5-flash"            # Gemini
# MODEL = "openai:gpt-4o"                      # OpenAI
# MODEL = "ollama:llama3"                      # Ollama (local)

response = client.chat.completions.create(
    model=MODEL,
    max_tokens=100,
    messages=[{"role": "user", "content": "Say 'tbh-code ready' and nothing else."}]
)
print(response.choices[0].message.content)
```

Run it:

```bash
python smoke_test.py
```

```
tbh-code ready
```

That's the abstraction at work. One client. One method. Swap the model string and your entire agent switches providers. The format is `provider:model-name` — aisuite routes to the right SDK automatically.

If you see `tbh-code ready`, your LLM connection works. If you get an authentication error, your API key isn't set. If you get a connection error, check your internet (or check that Ollama is running).

## The LLM Wrapper

Now create `tbh_code/llm.py`. This is the only file in your project that knows which provider you're using. Everything else calls this.

```python
# tbh_code/llm.py
import os
import aisuite as ai

MODEL = os.environ.get("TBH_MODEL", "anthropic:claude-sonnet-4-20250514")

_client = ai.Client()

def chat(messages, **kwargs):
    """Send messages to the LLM. Returns the assistant's response text."""
    response = _client.chat.completions.create(
        model=MODEL,
        messages=messages,
        **kwargs
    )
    return response.choices[0].message.content

def chat_raw(messages, **kwargs):
    """Send messages to the LLM. Returns the full response object.
    Use this when you need tool calls, usage stats, or structured output."""
    return _client.chat.completions.create(
        model=MODEL,
        messages=messages,
        **kwargs
    )
```

To switch providers, set one environment variable:

```bash
export TBH_MODEL="google:gemini-2.5-flash"            # Gemini
export TBH_MODEL="openai:gpt-4o"                       # OpenAI
export TBH_MODEL="ollama:llama3"                        # local Ollama
export TBH_MODEL="groq:llama-3.3-70b-versatile"        # Groq
```

Every chapter in this book calls `chat()` or `chat_raw()`. Your agent code never imports `anthropic`, `openai`, or `google` directly. The agent loop, tool calling, MCP, memory, planning, evaluation, the swarm — you build all of that on top of these two functions. aisuite is the foundation. Everything above it is yours.

Test it:

```python
# quick test — run from project root
from tbh_code.llm import chat

reply = chat([{"role": "user", "content": "Say 'llm wrapper works' and nothing else."}])
print(reply)
```
