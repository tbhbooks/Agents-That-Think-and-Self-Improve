================================================================================
KEY OBSERVATIONS FOR THE READER
================================================================================

1. ORCHESTRATOR-WORKERS — clean sequential flow, but a ceiling: bottleneck, fragile, rigid
2. FAN-OUT/FAN-IN — parallel processing, wall-clock = slowest agent (not sum)
3. PARTIAL RESULTS — when agents timeout, merge what you have + warnings
4. REVIEW CHAINS — pipelines emerge from individual routing decisions
5. CONSENSUS — weighted voting, domain expertise determines weight, dissent preserved
6. CONFLICT RESOLUTION — evidence first, then expert, then escalate
7. BACKPRESSURE — explicit capacity signals, requesters adapt, no silent failures
8. COLLECTIVE LEARNING — improvements propagate, bounce between agents, verified before adoption
9. ORCHESTRATOR → EVENTS — build the obvious thing, watch it break, replace with emergence
10. The bridge to Ch 14: the swarm works until an agent crashes, or you lose track of a multi-agent task, or a version upgrade breaks everything
