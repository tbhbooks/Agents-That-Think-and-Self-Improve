================================================================================
KEY OBSERVATIONS FOR THE READER
================================================================================

1. SKILLS ARE TOOLS — SkillTool has the same execute() -> ToolResult interface
2. SKILLS COMPOSE TOOLS — each step calls a tool from the registry
3. SKILL MATCHING is automatic — the agent picks the right skill for the task
4. STEP OUTPUTS FLOW — earlier step results feed into later step arguments
5. GRACEFUL FAILURE — missing tools cause clear errors, not silent failures
6. GRACEFUL FALLBACK — no skill match means general reasoning, not crash
7. THE REGISTRY IS UNIFIED — SimpleTools, MCPTools, and SkillTools coexist
8. Skills can REFERENCE tools that don't exist yet — the skill spec is a blueprint
9. The bridge to Ch 5 is visible: skills want to WRITE but can't yet
