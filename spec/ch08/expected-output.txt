================================================================================
KEY OBSERVATIONS FOR THE READER
================================================================================

1. SELF-EVALUATION — agent scores its own output on 3 criteria before presenting
2. GUARDRAILS — hard rules that block unconditionally (secrets, destructive commands)
3. FAIL-CLOSED — low scores trigger escalation, not silent delivery
4. ESCALATION — human gets context (what's done, what's wrong, options)
5. DIAGNOSTIC FEEDBACK — specific issues + actionable suggestions
6. GUARDRAILS vs GATES — guardrails are automatic (no prompt), gates ask permission
7. EVALUATION vs TESTING — tests check "does it run?", evaluation checks "is it good?"
8. The bridge to Ch 9: diagnostics are logged but don't change behavior — the agent will make the same mistakes again next session
