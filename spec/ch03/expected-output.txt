================================================================================
KEY OBSERVATIONS FOR THE READER
================================================================================

1. TOOL INTERFACE is uniform — read_file, list_files, search_code all return ToolResult
2. TOOL SELECTION — the agent picks the right tool for the job (search before read)
3. TOOL CHAINING — the agent uses multiple tools in sequence (search -> read -> verify)
4. VERIFICATION catches bad results — non-existent files, empty searches
5. MCP TOOLS are transparent — the agent doesn't know if a tool is local or remote
6. The agent is now ACTING, not just THINKING — it searches, reads, verifies
7. But it can't WRITE anything yet — that's Ch 5
8. And it uses tools chaotically — no strategy or playbook — that's Ch 4
