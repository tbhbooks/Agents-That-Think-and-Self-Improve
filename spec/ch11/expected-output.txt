================================================================================
KEY OBSERVATIONS FOR THE READER
================================================================================

1. BROADCAST BUS — file-based pub/sub, debuggable with ls and cat
2. AGENT CARDS — structured self-descriptions with capabilities AND skills
3. DISCOVERY PROTOCOL — subscribe + announce + read history (three steps)
4. PEER REGISTRY — queryable index: by name, capability, or skill
5. SKILL SHARING — skills spread only to tool-compatible agents
6. IMPROVEMENT SHARING — improvements propagate only when verified
7. WITHDRAWAL — clean shutdown removes from all peer registries
8. NO CENTRAL REGISTRY — each agent maintains its own peer list
9. IDEMPOTENT — re-announcing updates the existing entry, doesn't duplicate
10. The bridge to Ch 12: agents know WHO is out there, but can't TALK yet.
    Discovery is the address book. Communication is the conversation.
