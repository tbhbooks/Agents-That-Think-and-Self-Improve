================================================================================
KEY OBSERVATIONS FOR THE READER
================================================================================

1. AGENT IDENTITY — 8 fields define who an agent is. The system prompt IS the identity.
2. STRUCTURAL BOUNDARIES — tool lists enforce what agents can do, not just system prompts.
3. EFFORT BUDGETS — hard limits prevent runaway agents. Partial results on exhaustion.
4. SPECIALIZATION — same LLM, different system prompts = different specialists.
5. INDEPENDENT EVALUATION — reviewer catches what coder misses (no self-review bias).
6. CONTEXT ISOLATION — each agent's window holds only its role's artifacts.
7. GRACEFUL DEGRADATION — budget exhaustion returns partial results, not crashes.
8. The bridge to Ch 11: four agents, zero awareness of each other. The coder doesn't
   know the reviewer exists. Next: broadcast, discovery, and the first moment one agent
   learns that another exists.
